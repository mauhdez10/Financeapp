## 1. TL;DR

- **Verdict:** The brand, tokens, and intent are right; the implementation is a 7,950-line single-file monolith with two conflicting design systems pulling the UI back to AI-generic median every release.
- **Biggest single problem:** `src/App.jsx` is 7,950 lines (not 5,270 вЂ” it grew), 247 `useState`, 43 `useEffect`, ~140 components, with several full components minified onto one line. Babel has already deoptimized it. ESLint reports 268 errors including real bugs (`useTh` called conditionally, `Math.random` in render, duplicate translation keys silently dropping Spanish strings).
- **Chosen design direction:** **Editorial private-bank dark-mode default** вЂ” deep navy `#0D1B2A` + reserved gold `#C9A84C` accent + Newsreader italic for headlines and hero numbers + Plus Jakarta Sans for chrome + JetBrains Mono for tabular numerals. Hairline-divided sections, sharp corners (0вЂ“4px), no card-soup, no gradients. This is what `colors_and_type.css`, `SKILL.md`, `HANDOFF-v0.46.md`, and both `ui_kits/` already encode вЂ” every drift from it has been reverted (v0.34 orange/gold reverted in v0.47, cool slate reverted in v0.55).
- **Refactor approach:** **Strangler-fig by feature** вЂ” extract chart primitives в†’ modals в†’ pages в†’ sections out of App.jsx into `src/{tokens,primitives,charts,modals,features,pages}/`, each phase shippable on its own and verified by the existing Playwright smoke suite + visual diff against `preview/*.html`.
- **Do today:** Delete `design-system/golden-anchor/MASTER.md` (it's the AI-generic drift artifact вЂ” Calistoga + Inter + warm-stone + Dark-Mode-OLED, all contradicting the canonical CSS). Pin `colors_and_type.css` as the single source of truth. Patch the `xlsx` CVE (swap to `exceljs` or SheetJS CDN tarball). ## 4. Recommended Design Direction

**Direction:** **Editorial Private-Bank Dark Mode** вЂ” closer to *Mercury Г— Stripe Press Г— old Bessemer Trust quarterly* than to Robinhood or Linear. Default to deep navy, set hero numbers in Newsreader italic at 44px+, use gold sparingly as the brand accent (wordmark + one accent rule per page, never a button fill), strip Recharts defaults to one accent per chart, replace card-soup with hairline-divided sections, sharpen radii to 0вЂ“4px, force 64px desktop page margins.

**Evidence for this direction (from the owner's own work):**
- `SKILL.md` states it explicitly: *"dark-navy + rich-gold 'boutique private bank' feel"*.
- `colors_and_type.css` encodes 80% of these tokens already (navy/gold/cream-gold/Newsreader/Plus Jakarta/JetBrains Mono).
- `HANDOFF.md` + `HANDOFF-v0.46.md` lock the chart contract (thin strokes, JetBrains Mono numbers, gold caps titles).
- `ui_kits/advisor_app` and `ui_kits/client_portal` both ship dark-navy + gold.
- Every drift from this has been reverted: v0.34 orange/gold в†’ v0.47 red/green; cool slate в†’ v0.55 warm linen; v0.57 sign-in WCAG fix (cream-on-gold failed AA at 2.2:1 в†’ walnut + cream text 7.7:1 AAA).
- v0.42 CHANGELOG: *"shift away from chunky/saturated toward modern fintech (Linear / Robinhood)"* вЂ” the direction is locked in writing.

**Why this grouping:**

The split is **by feature, not by layer** at the top вЂ” `features/client`, `features/intake`, `features/calculators` map to user-visible product surfaces, so a future "remove the calculator section" or "rebuild intake" is one folder. Within each feature, layers (`tabs/`, `sections/`, hooks) emerge naturally. Primitives, charts, modals, and pages sit at the top because they're shared across features.

The owner has stated (D-1) that single-file architecture aids his GitHub-web-UI workflow. This layout **preserves the same property at the *file* level**: each file is small enough to upload through the web UI, and `translations.js` + `engagementLetterTemplate.js` stay exactly where they are. The owner's "I want to edit one file" instinct is honored вЂ” but now that file is a 150-line `shell/App.tsx`, not a 7,950-line everything.

The `shell/App.tsx` is the explicit auth state machine the current four-conditional micro-router pretends to be. It owns nothing but routing and auth bootstrap. All state lives in `features/*/use*.ts` hooks or in the new `SettingsCtx`/`TranslationCtx`. This is what eliminates the 247-useState problem without introducing Redux/Zustand (still bound by D-7).

## 8. What NOT to Touch

These are locked in `AGENT.md` and any session that violates them is wrong:

- **D-1 Single-file ergonomic preserved.** The owner edits via GitHub web UI. After refactor, each file must remain small and self-contained; do not introduce a build system the owner can't navigate. Web-UI uploads must still work file-by-file.
- **D-4 MutationObserver-based DOM translation is BANNED.** Caused infinite re-render loop. Never reintroduce.
- **D-7 No Redux / Zustand / Jotai / Recoil.** Only `useState`/`useRef`/`useMemo` + React Context. The new `SettingsCtx` and `TranslationCtx` stay within this rule.
- **D-8 Recharts only** for charts (no Chart.js / D3 / Highcharts). The new SVG primitives are pure-React SVG, not a chart library вЂ” consistent with this rule.
- **D-9 xlsx (SheetJS)** is the locked Excel I/O. If swapping to `exceljs` to close the CVE, get explicit owner approval вЂ” otherwise pin SheetJS CDN tarball.
- **D-11 Component naming PascalCase.** Dynamic refs must rebind (`const Comp = calc.C`). JSX treats lowercase as HTML.
- **D-12 React hook imports** at top of every file that uses them.
- **D-18 Translation Track A + Track B.** Don't reshape data structures for translation; per-key wrap `t.key||"English"`.
- **D-23 Multi-tenant via RLS only** вЂ” not per-customer schema duplication.
- **D-27 Mobile-first responsive shell, centered modals on mobile** (not bottom-sheet вЂ” bottom-sheet rejected v0.13.2).
- **D-36 Bulk static-text patches MUST be scope-aware** вЂ” never run global `text.replace("WORD", ...)`. Anchor to JSX context.
- **Pitfall: lowercase dynamic JSX components render as HTML.** Always rebind.
- **Pitfall: Calculator components without `{t}` param cause silent ReferenceError + black screen.**
- **Pitfall: BOTH languages, ALWAYS.** Every user-facing string lands in `T.en` AND `T.es` in the same edit.
- **Pitfall: CSS `zoom` traps `position:fixed` descendants in WebKit/iOS Safari.** Modals/overlays/toasts must be siblings of the zoom-carrying container.
- **Pitfall: Don't change data-structure shape for translation.** Use D-18 Track B (`ACCT_META`, `LOAN_META`, `PHYS_CATS` referenced in 25+ places).
- **Pitfall: URL-based routing inside one React function component** вЂ” route check must come AFTER all hook declarations.
- **Pitfall: PostgREST `.or()` filter cannot contain dotted values.** Use separate `.eq()` calls.
- **Pitfall: in-app nav must push browser history** with first-render `replaceState` + `_popstateRestoringRef` guard.
- **Open O-12 auto-logout 30min idle + 1min warning** вЂ” keep.
